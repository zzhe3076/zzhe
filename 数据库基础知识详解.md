## 第一章：数据库基本概念

### 1.1 什么是数据库？

**数据库**是存储数据的"电子仓库"。

```
传统纸质档案:              数据库:
┌─────────────┐          ┌──────────────┐
│  档案室     │    →    │  数据库      │
│  找文件慢   │          │  找数据快   │
│  占用空间大 │          │  占用空间小  │
└─────────────┘          └──────────────┘
```

### 1.2 为什么需要数据库？

| 问题 | 解决方案 |
|:---|:---|
| 文件查找慢 | 数据库索引，快速定位 |
| 数据冗余 | 统一存储，避免重复 |
| 数据不一致 | 集中管理，保证一致 |
| 并发访问 | 多用户同时访问 |
| 数据安全 | 权限控制、备份恢复 |

## 第二章：核心术语

### 2.1 表（Table）

**表 = Excel 表格**

```
┌─────────────────────────────────────────────────────┐
│                   student (学生表)                  │
├────┬────────┬────────┬──────────┬───────────────┤
│ id │ name   │ gender │ phone    │ status        │

**记录 = 表中的一行数据**

```

{id:1, name:"张三", gender:"男", phone:"138...", status:"pending"}

## 第四章：主键与外键

### 4.1 主键（Primary Key）

**主键 = 唯一标识每条记录的"身份证号"**

```sql
-- id 是主键，唯一且非空
CREATE TABLE student (
    id SERIAL PRIMARY KEY,    -- 主键
    name VARCHAR(50)
);
```

**特点**：
- ✅ 唯一（不能重复）
- ✅ 非空（必须有值）
- ✅ 每个表只有一个

### 4.2 外键（Foreign Key）

**外键 = 表与表之间的"联系方式"**

```sql
-- student 表的 major_id 引用 major 表的 id
CREATE TABLE student (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50),
    major_id INT REFERENCES major(id)  -- 外键
);
```

**图示**：

```
┌─────────────┐         ┌─────────────┐
│   major     │         │   student   │
│   专业表     │         │   学生表    │
├─────────────┤         ├─────────────┤
│ id (主键)   │◄────────│ major_id   │
│ name        │  外键   │ (外键)     │
└─────────────┘         └─────────────┘
```

## 第五章：表关系

### 5.1 一对多（1:N）

**最常见的关​​系**

```
1个学院  ──────→  多个专业
1个专业  ──────→  多个学生
1个学生  ──────→  多个缴费记录
```

**代码实现**：

```python
class College(models.Model):
    name = models.CharField(max_length=100)

class Student(models.Model):
    name = models.CharField(max_length=50)
    # ForeignKey = 一对多
    college = models.ForeignKey(College, on_delete=models.CASCADE)
```

### 5.2 一对一（1:1）

**特殊的一对关系**

```
1个用户  ──────  1个学生
```

**代码实现**：

```python
class User(models.Model):
    username = models.CharField(max_length=50)

class Student(models.Model):
    name = models.CharField(max_length=50)
    # OneToOneField = 一对一
    user = models.OneToOneField(User, on_delete=models.CASCADE)
```

### 5.3 多对多（N:N）

**双向一对多**

```
多个学生  ←────→  多个课程
```

**代码实现**：

```python
class Student(models.Model):
    name = models.CharField(max_length=50)

class Course(models.Model):
    name = models.CharField(max_length=100)
    # ManyToManyField = 多对多
    students = models.ManyToManyField(Student)
```

## 第六章：SQL 基础语句

### 6.1 INSERT - 插入数据

```sql
-- 插入一条
INSERT INTO student (name, gender, phone)
VALUES ('张三', '男', '13800138001');

-- 插入多条
INSERT INTO student (name, gender) VALUES
    ('李四', '女'),
    ('王五', '男');
```

### 6.2 SELECT - 查询数据

```sql
-- 查所有
SELECT * FROM student;

-- 查指定列
SELECT name, phone FROM student;

-- 条件查询
SELECT * FROM student WHERE status = 'pending';

-- 排序
SELECT * FROM student ORDER BY created_at DESC;

-- 限制数量
SELECT * FROM student LIMIT 10;

-- 分页 (跳过前5条，取10条)
SELECT * FROM student LIMIT 10 OFFSET 5;
```

### 6.3 UPDATE - 更新数据

```sql
-- 更新单条
UPDATE student
SET status = 'checked_in'
WHERE id = 1;

-- 批量更新
UPDATE student
SET status = 'checked_in'
WHERE status = 'pending';
```

### 6.4 DELETE - 删除数据

```sql
-- 删除单条
DELETE FROM student WHERE id = 1;

-- 条件删除
DELETE FROM student WHERE status = 'cancelled';

-- 删除全部
DELETE FROM student;
```

## 第八章：表连接（JOIN）

### 8.1 什么是连接？

把多个表的数据合并在一起查询。

```
student表                    major表
┌──────────────┐           ┌──────────┐
│ id=1, major=1│           │ id=1     │
│ id=2, major=1│    JOIN   │ name=软件 │
└──────────────┘           └──────────┘
              ↓
┌────────────────────────────────┐
│ name | student_id | major_name │
│ 张三  │ 001        │ 软件工程  │
│ 李四  │ 002        │ 软件工程  │
└────────────────────────────────┘
```

### 8.2 INNER JOIN - 内连接

```sql
-- 查学生和专业（只显示有专业的学生）
SELECT s.name, m.name as major_name
FROM student s
INNER JOIN major m ON s.major_id = m.id;
```

### 8.3 LEFT JOIN - 左连接

```sql
-- 查所有学生，即使没有专业也显示
SELECT s.name, m.name as major_name
FROM student s
LEFT JOIN major m ON s.major_id = m.id;
```

## 第十章：索引

### 10.1 什么是索引？

**索引 = 书的目录**

```
无索引: 逐行查找         有索引: 直接定位
┌─────────────┐         ┌─────────────┐
│ 查找"张三"  │         │ 查找"张三"  │
│ ████████████│         │ ░░░░░░█░░░░░│
│ 逐行扫描    │         │ 直接定位    │
└─────────────┘         └─────────────┘
  慢: O(n)               快: O(log n)
```

### 10.2 创建索引

```sql
-- 单列索引
CREATE INDEX idx_student_status ON student(status);

-- 复合索引 (同时查多个字段)
CREATE INDEX idx_student_status_major ON student(status, major_id);

-- 唯一索引
CREATE UNIQUE INDEX idx_student_id ON student(student_id);
```

### 10.3 何时加索引？

| 场景 | 建议 |
|:---|:---|
| WHERE 条件字段 | ✅ 加索引 |
| JOIN 连接字段 | ✅ 加索引 |
| ORDER BY 排序字段 | ✅ 加索引 |
| 区分度低的字段 | ❌ 不加 |
| 频繁更新的字段 | ❌ 不加 |

## 第十一章：事务

### 11.1 什么是事务？

**事务 = 打包执行，要么全成功，要么全失败**

```
例子: 转账
  A账户 -100元  →  成功
  B账户 +100元  →  成功
  ↓
  提交事务

例子: 失败情况
  A账户 -100元  →  成功
  B账户 +100元  →  失败
  ↓
  回滚事务 (A账户也恢复)
```

### 11.2 事务特性（ACID）

| 特性 | 说明 |
|:---|:---|
| Atomic | 原子性：要么全成功，要么全失败 |
| Consistent | 一致性：事务前后数据一致 |
| Isolation | 隔离性：并发事务互不干扰 |
| Durable | 持久性：提交后数据永久保存 |

### 11.3 使用示例

```sql
BEGIN;  -- 开始事务

INSERT INTO student (name) VALUES ('张三');
INSERT INTO check_in (student_id) VALUES (1);

COMMIT;  -- 提交
-- 或 ROLLBACK;  -- 回滚
```

## 第十三章：Django ORM 操作

### 13.1 模型定义

```python
from django.db import models

class College(models.Model):
    name = models.CharField(max_length=100)  # 字符串
    code = models.CharField(max_length=20, unique=True)  # 唯一
    description = models.TextField()  # 长文本

    class Meta:
        db_table = 'college'  # 表名

    def __str__(self):
        return self.name
```

### 13.2 CRUD 操作

```python
# ====== 增 ======
# 方式1
college = College(name='计算机学院', code='CS')
college.save()

# 方式2
College.objects.create(name='软件学院', code='SE')


# ====== 查 ======
# 查所有
colleges = College.objects.all()

# 条件查
college = College.objects.get(id=1)  # 查一个
colleges = College.objects.filter(code='CS')  # 条件查

# 排序
colleges = College.objects.order_by('-code')  # 降序


# ====== 改 ======
college = College.objects.get(id=1)
college.name = '新名字'
college.save()

# 批量改
College.objects.filter(code='CS').update(name='新学院')


# ====== 删 ======
college = College.objects.get(id=1)
college.delete()

# 批量删
College.objects.filter(code='CS').delete()
```

### 13.3 外键查询

```python
# 查学生所属专业
student = Student.objects.get(id=1)
major = student.major  # 自动查询
print(major.name)

# 查专业有哪些学生
major = Major.objects.get(id=1)
students = major.student_set.all()
```

### 13.4 关联查询

```python
# 跨表查询：查北京的学生
students = Student.objects.filter(origin_province='北京')

# 跨表查询：查计算机学院的专业
majors = Major.objects.filter(college__name='计算机学院')
# __ 表示跨表
```

## 第十四章：常见问题

### 14.1 什么是 N+1 问题？

```python
# ❌ 慢: 查10个学生，每个学生又查1次专业 = 11次查询
students = Student.objects.all()
for s in students:
    print(s.major.name)  # 每次都会查数据库

# ✅ 快: 1次查询搞定
students = Student.objects.select_related('major').all()
for s in students:
    print(s.major.name)  # 不额外查询
```

### 14.2 什么是 SQL 注入？

```python
# ❌ 危险: 直接拼接字符串
name = "'; DELETE FROM student; --"
Student.objects.filter(name=name)  # 会被攻击!

# ✅ 安全: 使用参数化查询
Student.objects.filter(name=name)  # Django 自动处理
```
