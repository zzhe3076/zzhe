# 数据库完全指南

## 一、数据库简介

### 1.1 什么是数据库？

数据库是存储在计算机中的结构化数据集合，可以高效地存储、检索和管理数据。

```
未使用数据库:                    使用数据库:
┌─────────┐                    ┌──────────────┐
│ 文件1   │                    │   数据库      │
│ 文件2   │  ← 难以管理  ←    │  ┌────────┐  │
│ 文件3   │                    │  │ 表1    │  │
│ Excel   │                    │  ├────────┤  │
│ CSV     │                    │  │ 表2    │  │
└─────────┘                    │  ├────────┤  │
                              │  │ 表3    │  │
                              │  └────────┘  │
                              └──────────────┘
```

### 1.2 数据库类型

| 类型 | 说明 | 典型代表 |
|:---|:---|:---|
| 关系型(RDBMS) | 用表存储数据，有行列 | MySQL, PostgreSQL, Oracle |
| 非关系型(NoSQL) | 键值、文档、列族存储 | MongoDB, Redis, Cassandra |
| 面向对象 | 用对象存储数据 | ObjectDB |

### 1.3 本项目使用 PostgreSQL

PostgreSQL 是一个功能强大的开源对象关系型数据库：

- ✅ 开源免费
- ✅ 支持复杂查询
- ✅ 事务支持
- ✅ JSON 支持
- ✅ 性能优秀

## 二、数据库核心概念

### 2.1 表（Table）

表是数据库中最基本的结构，由行和列组成：

```
┌─────────────────────────────────────────────────────┐
│                    student 表                        │
├─────┬────────┬────────┬─────────┬────────┬──────┤
│  id │ name   │ gender │ phone   │ status│ major│
├─────┼────────┼────────┼─────────┼────────┼──────┤
│  1  │ 张三   │ 男     │ 138...  │ 已报到 │ 1    │
│  2  │ 李四   │ 女     │ 139...  │ 待报到 │ 1    │
│  3  │ 王五   │ 男     │ 137...  │ 已报到 │ 2    │
└─────┴────────┴────────┴─────────┴────────┴──────┘
   ↑                   ↑
   行(Row)            列(Column)
```

### 2.2 字段类型

| 类型 | 说明 | 示例 |
|:---|:---|:---|
| INT/INTEGER | 整数 | 1, 100, -5 |
| VARCHAR(n) | 可变字符串 | '张三' |
| TEXT | 长文本 | 文章内容 |
| DECIMAL(p,s) | 精确小数 | 99.99 |
| DATE | 日期 | 2024-09-01 |
| DATETIME | 日期时间 | 2024-09-01 10:30:00 |
| BOOLEAN | 布尔值 | TRUE/FALSE |
| JSON | JSON数据 | {"key": "value"} |

### 2.3 主键 (Primary Key)

主键是唯一标识表中每行数据的字段：

```sql
-- id 是主键，唯一且非空
CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);
```

### 2.4 外键 (Foreign Key)

外键用于建立表与表之间的关系：

```sql
-- student 表的 major_id 引用 major 表的 id
CREATE TABLE student (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    major_id INT REFERENCES major(id)
);
```

## 三、SQL 详细讲解

### 3.1 DDL - 数据定义语言

创建、修改、删除数据库对象

#### 创建表

```sql
CREATE TABLE student (
    id SERIAL PRIMARY KEY,          -- 自增主键
    name VARCHAR(50) NOT NULL,      -- 非空
    student_id VARCHAR(20) UNIQUE,  -- 唯一
    phone VARCHAR(20),
    status VARCHAR(20) DEFAULT 'pending',  -- 默认值
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP  -- 默认时间
);
```

#### 修改表

```sql
-- 添加字段
ALTER TABLE student ADD COLUMN email VARCHAR(100);

-- 修改字段
ALTER TABLE student ALTER COLUMN phone TYPE VARCHAR(15);

-- 删除字段
ALTER TABLE student DROP COLUMN email;

-- 重命名表
ALTER TABLE student RENAME TO students;
```

#### 删除表

```sql
-- 删除表（谨慎！）
DROP TABLE student;

-- 清空表数据
TRUNCATE TABLE student RESTART IDENTITY;
```

### 3.2 DML - 数据操作语言

#### INSERT - 插入数据

```sql
-- 插入单条
INSERT INTO student (name, student_id, phone)
VALUES ('张三', '2024001', '13800138001');

-- 插入多条
INSERT INTO student (name, student_id) VALUES
    ('李四', '2024002'),
    ('王五', '2024003'),
    ('赵六', '2024004');

-- 从其他表插入
INSERT INTO student_backup (name, student_id)
SELECT name, student_id FROM student;
```

#### UPDATE - 更新数据

```sql
-- 更新单条
UPDATE student
SET phone = '13900139001', status = 'checked_in'
WHERE id = 1;

-- 批量更新
UPDATE student
SET status = 'checked_in'
WHERE created_at < '2024-09-01';

-- 使用表达式
UPDATE student
SET check_in_time = NOW()
WHERE status = 'pending';
```

#### DELETE - 删除数据

```sql
-- 删除单条
DELETE FROM student WHERE id = 1;

-- 条件删除
DELETE FROM student WHERE status = 'cancelled';

-- 删除全部（谨慎！）
DELETE FROM student;
```

### 3.3 DQL - 数据查询语言

#### 基本查询

```sql
-- 查询所有字段
SELECT * FROM student;

-- 查询指定字段
SELECT name, student_id, phone FROM student;

-- 字段别名
SELECT name AS 姓名, student_id AS 学号 FROM student;

-- 去重
SELECT DISTINCT status FROM student;
```

#### 条件查询 WHERE

```sql
-- 等于
SELECT * FROM student WHERE status = 'pending';

-- 不等于
SELECT * FROM student WHERE status != 'checked_in';

-- 大于/小于
SELECT * FROM student WHERE id > 10;

-- 范围
SELECT * FROM student WHERE id BETWEEN 1 AND 10;

-- 多个条件
SELECT * FROM student
WHERE status = 'pending' AND major_id = 1;

-- IN 查询
SELECT * FROM student WHERE major_id IN (1, 2, 3);

-- LIKE 模糊查询
SELECT * FROM student WHERE name LIKE '张%';    -- 张开头
SELECT * FROM student WHERE name LIKE '%张%';   -- 包含张
SELECT * FROM student WHERE name LIKE '_三';   -- 第二个字是三
```

#### 排序 ORDER BY

```sql
-- 升序（默认）
SELECT * FROM student ORDER BY created_at;

-- 降序
SELECT * FROM student ORDER BY created_at DESC;

-- 多字段排序
SELECT * FROM student
ORDER BY status ASC, created_at DESC;
```

#### 限制结果 LIMIT

```sql
-- 前10条
SELECT * FROM student LIMIT 10;

-- 跳过前5条，取10条（分页）
SELECT * FROM student LIMIT 10 OFFSET 5;

-- 等同于
SELECT * FROM student LIMIT 5, 10;
```

#### 聚合函数

| 函数 | 说明 | 示例 |
|:---|:---|:---|
| COUNT() | 计数 | COUNT(*) |
| SUM() | 求和 | SUM(amount) |
| AVG() | 平均 | AVG(age) |
| MAX() | 最大 | MAX(score) |
| MIN() | 最小 | MIN(score) |

```sql
-- 统计总数
SELECT COUNT(*) FROM student;

-- 统计已报到人数
SELECT COUNT(*) FROM student WHERE status = 'checked_in';

-- 分组统计
SELECT status, COUNT(*) as 人数
FROM student
GROUP BY status;

-- 分组后筛选
SELECT status, COUNT(*) as 人数
FROM student
GROUP BY status
HAVING COUNT(*) > 10;
```

#### 表连接 JOIN

```sql
-- INNER JOIN 内连接
SELECT s.name, m.name as 专业
FROM student s
INNER JOIN major m ON s.major_id = m.id;

-- LEFT JOIN 左连接
SELECT s.name, m.name as 专业
FROM student s
LEFT JOIN major m ON s.major_id = m.id;

-- 多表连接
SELECT s.name, m.name as 专业, c.name as 学院
FROM student s
JOIN major m ON s.major_id = m.id
JOIN college c ON m.college_id = c.id;
```

## 四、数据库设计

### 4.1 规范化

数据库规范化减少数据冗余：

```
第一范式(1NF): 字段原子性
┌─────────────────────┐
│  错误设计            │
│  地址: 北京朝阳区XX  │
└─────────────────────┘
         ↓
┌─────────────────────┐
│  正确设计            │
│  省份: 北京          │
│  城市: 北京          │
│  区县: 朝阳区        │
└─────────────────────┘

第二范式(2NF): 消除部分依赖
学生(学号, 姓名, 专业编号, 专业名称)
         ↓ 拆分
学生(学号, 姓名, 专业编号)
专业(专业编号, 专业名称)

第三范式(3NF): 消除传递依赖
学生(学号, 姓名, 学院编号, 学院名称)
         ↓ 拆分
学生(学号, 姓名, 学院编号)
学院(学院编号, 学院名称)
```

## 五、索引

### 5.1 索引原理

索引就像书的目录，加快查询速度：

```
无索引: 全表扫描                    有索引: 快速定位
┌─────────────┐                   ┌─────────────┐
│ 查找"张三"  │                   │ 查找"张三"  │
│ ████████████│                   │ ░░░░░░░█░░░░│
│ ████████████│                   │ 快速定位    │
└─────────────┘                   └─────────────┘
  慢: O(n)                         快: O(log n)
```

### 5.2 创建索引

```sql
-- 单列索引
CREATE INDEX idx_student_status ON student(status);

-- 复合索引
CREATE INDEX idx_student_status_major ON student(status, major_id);

-- 唯一索引
CREATE UNIQUE INDEX idx_student_id ON student(student_id);

-- 查看索引
\d student
```

### 5.3 索引使用场景

| 场景 | 推荐 |
|:---|:---|
| WHERE 条件字段 | ✅ 加索引 |
| JOIN 连接字段 | ✅ 加索引 |
| ORDER BY 排序字段 | ✅ 加索引 |
| 区分度低的字段 | ❌ 不加 |
| 频繁更新的字段 | ❌ 不加 |

## 六、事务

### 6.1 事务特性 ACID

| 特性 | 说明 |
|:---|:---|
| Atomic(原子性) | 要么全成功，要么全失败 |
| Consistency(一致性) | 事务前后数据一致 |
| Isolation(隔离性) | 并发事务互不干扰 |
| Durability(持久性) | 提交后数据永久保存 |

### 6.2 事务示例

```sql
BEGIN;  -- 开始事务

-- 操作1: 创建学生
INSERT INTO student (name, student_id)
VALUES ('张三', '2024001');

-- 操作2: 创建报到记录
INSERT INTO check_in (student_id, method)
VALUES (1, 'manual');

-- 操作3: 更新状态
UPDATE student SET status = 'checked_in' WHERE id = 1;

COMMIT;  -- 提交事务
-- 或 ROLLBACK;  -- 回滚事务
```

### 6.3 Django 事务

```python
from django.db import transaction

# 方式1: 上下文管理器
with transaction.atomic():
    student = Student.objects.create(...)
    CheckIn.objects.create(student=student, ...)

# 方式2: 装饰器
@transaction.atomic
def process_checkin(request):
    ...
```

## 八、存储过程

存储过程是预编译的SQL语句集合：

```sql
-- 创建存储过程
CREATE OR REPLACE FUNCTION get_student_count(p_status VARCHAR)
RETURNS INT AS $$
DECLARE
    count_val INT;
BEGIN
    SELECT COUNT(*) INTO count_val
    FROM student
    WHERE status = p_status;
    RETURN count_val;
END;
$$ LANGUAGE plpgsql;

-- 调用
SELECT get_student_count('pending');
```

## 十、Django ORM 进阶

### 10.1 模型定义

```python
from django.db import models

class College(models.Model):
    name = models.CharField(max_length=100)
    code = models.CharField(max_length=20, unique=True)

    class Meta:
        db_table = 'college'
        ordering = ['code']

    def __str__(self):
        return self.name

class Student(models.Model):
    GENDER_CHOICES = [
        ('male', '男'),
        ('female', '女'),
    ]

    name = models.CharField(max_length=50)
    student_id = models.CharField(max_length=20, unique=True)
    gender = models.CharField(max_length=10, choices=GENDER_CHOICES)
    major = models.ForeignKey(Major, on_delete=models.CASCADE)
    status = models.CharField(max_length=20, default='pending')

    class Meta:
        db_table = 'student'
        indexes = [
            models.Index(fields=['status', 'major']),
        ]
```

### 10.2 查询操作

```python
# 增
student = Student.objects.create(
    name='张三',
    student_id='2024001'
)

# 查 - 单条
student = Student.objects.get(id=1)  # 不存在抛异常
student = Student.objects.filter(id=1).first()  # 安全获取

# 查 - 列表
students = Student.objects.all()
students = Student.objects.filter(status='pending')
students = Student.objects.exclude(status='cancelled')

# 链式查询
students = Student.objects.filter(
    status='pending'
).filter(
    major__college_id=1
).order_by('-created_at')[:10]

# 关联查询
students = Student.objects.select_related('major__college').all()

# 聚合
from django.db.models import Count, Avg, Sum
stats = Student.objects.aggregate(
    total=Count('id'),
    checked_in=Count('id', filter=Q(status='checked_in'))
)

# 分组
from django.db.models import Count
colleges = College.objects.annotate(
    student_count=Count('student')
)
```

### 10.3 更新和删除

```python

# 更新多条

student.delete()
```

### 练习2: 查询统计

```sql
-- 1. 统计各状态学生数
SELECT status, COUNT(*) as 人数
FROM student
GROUP BY status;

-- 2. 统计各专业学生数
SELECT m.name as 专业, COUNT(s.id) as 人数
FROM student s
JOIN major m ON s.major_id = m.id
GROUP BY m.id, m.name
ORDER BY 人数 DESC;

-- 3. 查询未报到学生
SELECT name, student_id, phone
FROM student
WHERE status = 'pending'
ORDER BY created_at;
```
