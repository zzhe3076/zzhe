# 大学生新生报到系统 - 技术学习方案

## 一、学习目标

通过本学习方案，掌握开发「大学生新生报到智能助手」所需的全栈技术能力，能够独立完成前后端开发、数据库设计、API接口开发、微信小程序开发及项目部署。

## 十四、学习建议

### 14.1 核心原则
1. **先跑通再优化**：先实现功能，再考虑性能优化
2. **实践为主**：只看不动手 = 不会，每学必练
3. **循序渐进**：不要跳过基础直接学高级内容
4. **复盘总结**：每完成一个模块，写技术笔记

### 14.2 时间安排
- 每天学习 2-3 小时
- 周末可适当增加时间
- 每两周完成一个阶段
- 遇到问题先自行搜索，再请教他人

### 14.3 代码管理
- 使用 Git 管理代码版本
- 提交信息要清晰规范
- 定期推送到 GitHub
